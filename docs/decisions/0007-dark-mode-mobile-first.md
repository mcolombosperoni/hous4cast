# 0007 - Dark mode and mobile-first

- Status: accepted
- Date: 2026-04-25

## Context
The product target is QR-first, therefore most traffic will be mobile.

## Decision
Adopt mobile-first layouts and support explicit dark mode toggle with persisted preference.

## Alternatives considered
- Desktop-first and no theme toggle: rejected for UX goals.

## Consequences
- Better mobile usability.
- Design token discipline required from start.
